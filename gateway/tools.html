CRUD & Search

Full XML-RPC / JSON-RPC coverage with live refresh bus.

odoo_search

returns: list[int]

Search for record IDs matching a domain filter.

Parameters & use cases
Parameters
  • model — model name (e.g. 'res.partner')
  • domain — Odoo domain list
  • limit, offset, order
Use cases
  • "Find all draft invoices for partner X"
  • "Last 10 sale orders posted this month"
  • "All active users with stock group"
Example
odoo_search(model='res.partner',
  domain=[['is_company','=',true],['country_id.code','=','BG']],
  limit=50, order='name')

Translations NEW 2.10

Multi-language field management — version-aware, auto-ZWSP for identical values.

4 tools

odoo_list_translatable_fields

inspector

Discovers which fields on a model are translatable and classifies each.

Parameters & use cases
Classifications
  • simple — translate=True (Char/Text/Selection)
  • html — translate=html_translate (blog.post.content, product.website_description)
  • xml — translate=xml_translate (ir.ui.view.arch_db)
  • callable — other truthy (custom translate function)
Use cases
  • "Which fields on blog.post can I translate?"
  • "Find all html_translate fields on product.template"
odoo_list_translatable_fields(model='blog.post')
→ 9 fields: name, subtitle, teaser, teaser_manual (simple);
  content (html); website_meta_title, seo_name... (simple)

odoo_get_field_translations

reader

Read current per-lang translations for a field. Auto-detects kind and uses the right API (16+ native JSONB, <16 ir.translation fallback).

odoo_translate_field

simple · write

Writes translations for simple translate=True fields (name, subtitle, teaser). Validates lang activation. Version-oriented.

Use cases
  • "Set Bulgarian translation for blog post title"
  • "Translate product name to 3 languages at once"
  • "Clear a stale translation (pass null)"
odoo_translate_field(model='blog.post', res_id=180,
  field_name='subtitle',
  translations={'bg_BG': 'Как премахването влияе върху счетоводството'})

odoo_translate_html

html/xml · 3 modes

Writes translations for html_translate / xml_translate fields. Three modes:

Modes & use cases
Modes
  • extract — read-only; returns canonical terms as Odoo's engine sees them (HTML blocks with inline tags preserved)
  • terms — direct {lang: {src_term: tr_term}} map; precise
  • replace — {lang: full_html}; delegated to Odoo ORM via write() with lang context
Auto-ZWSP fix

When a target-lang value equals the source, Odoo silently drops it (treated as "untranslated"). This tool automatically prefixes U+200B (zero-width-space) to keep it stored as "translated, kept intentionally" — avoiding the website editor's "translate" indicator.

Use cases
  • "Translate a full blog article keeping existing HTML structure"
  • "Swap one specific term across 3 languages"
  • "Extract what terms need translation before sending to a translator"
odoo_translate_html(model='blog.post', res_id=180,
  field_name='content', mode='replace',
  translations={'bg_BG': '<section>...full BG html...</section>'})

Website Snippets NEW 2.10

lxml-based HTML mutation with xpath substitutions for blog posts, pages, product descriptions, menus.

5 tools

odoo_website_list_snippets

List available Odoo snippet templates (ir.ui.view with key like website.s_*).

Use cases & categories
Categories inferred
  • structure — s_banner, s_kickoff, s_cover, s_intro
  • content — s_text_block, s_cards_grid, s_three_columns, s_cta_card
  • dynamic — s_dynamic_snippet_*, s_blog_posts, s_products
  • effect — s_progress_bar, s_countdown, s_parallax
Filters
  • category — filter by inferred category
  • module — website, website_blog, website_sale, website_forum
  • search — substring in key or name

odoo_website_list_page_snippets

List snippets currently on a target HTML field. Parses with lxml. Detects by both data-snippet attribute AND first s_* class (Odoo strips data-snippet on some saves).

Returns
  • index, xpath, data_snippet, data_name
  • text_preview — first 150 chars of visible text
  • has_background, background_url — detected from style attribute

odoo_website_add_snippet

write

Fetches snippet arch from ir.ui.view, extracts root element, applies pre-insertion substitutions, inserts at position.

Positions & examples
Positions
  • end — after last element (default)
  • begin — before first element
  • after / before / replace — relative to anchor_xpath
Example — add CTA at end with custom heading
odoo_website_add_snippet(
  model='blog.post', res_id=180, field_name='content',
  snippet_key='website.s_cta_card', position='end',
  substitutions={
    './/h2': 'Need help with your Odoo migration?',
    ".//a[contains(@class,'btn')]/@href": 'https://yoursite.com/contact',
  })

odoo_website_update_snippet

write

Update existing snippet content on target. Locates via snippet_xpath, applies substitutions relative to it.

Substitution syntax
  • {'.//h2': 'New Title'} — text replace
  • {'.//img/@src': 'https://...'} — attribute set
  • {'./div/@style:background-image': 'url(...)'} — single CSS property (preserves other style props)
Example — swap banner background
odoo_website_update_snippet(
  model='blog.post', res_id=180, field_name='content',
  snippet_xpath='/div/section[1]',
  substitutions={
    './div/@style:background-image': 'url(/web/image/67341)',
    './div/@style:background-size': 'cover',
    './div/@style:min-height': '420px',
  })

odoo_website_remove_snippet

write

Remove snippet at xpath. Returns preview of removed HTML for audit.
