WHY THIS EXISTS

Single-purpose MCP wrappers don't scale.
This one is built for real production.

01

Unified authentication

Token-based auth across all MCP endpoints. One identity, all servers. Per-user profile isolation.

02

Multi-tenant by design

One stack, dozens of Odoo databases. Switch connections on the fly. Each user sees only their own data.

03

Claude.ai connector ready

Public HTTPS endpoint with bearer token auth. Register as a Custom Connector in Team or Enterprise.

04

K3s & Docker Compose

Kustomize overlays for dev and prod. Docker Compose for solo developers. Pick your scale.

05

Browser-based terminal

xterm.js + tmux + Claude Code in the browser. Themable, persistent sessions, auth-integrated.

06

Vector store native

Qdrant integration for semantic search across Odoo records. Ollama for local LLM inference.

GET STARTED

Three ways to deploy.

01

Docker Compose

Local development 
# Clone & configure
$ git clone https://github.com/rosenvladimirov/odoo-claude-mcp.git
$ cd odoo-claude-mcp
$ cp .env.example .env
$ nano .env

# Start the stack
$ docker compose up -d
✓ 8 services running
02

Install script

Quickest path 
# Linux / macOS
$ curl -fsSL raw.../install.sh | bash

# Windows (PowerShell admin)
> iwr -useb raw.../install.ps1 | iex

✓ Docker containers pulled
✓ Env configured interactively
✓ Stack up on localhost:8084
03

Kubernetes / K3s

Production 
# Kustomize overlays in k3s/
$ cd k3s/overlays/prod
$ cp .env.example .env
$ cp ../../base/secrets.example.yaml secrets.yaml

# Edit secrets, then deploy
$ kubectl apply -k .
✓ Deployed with cert-manager + ingress
